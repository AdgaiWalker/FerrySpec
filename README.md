## 这是什么

`Ferry` 是一套面向人机协作的世界协议。
它试图回答的不是“再做一个 AI 工具”，而是：

- 人的意图，怎样稳定地落到现实产出里
- AI 的执行，怎样不越过人的价值判断
- 一个项目、一次对话、一套方法，怎样不在下一次协作里断掉

它把这件事抽象成一个最小框架：

- `Skill`：定规矩
- `Flow`：干活
- `Log`：进化

一句话理解：

**Ferry 想做的，是把“蓝图、现实、记忆”重新接成一条可持续的人机协作链。**

## 它解决什么问题

很多协作问题，不是因为不会做，而是因为这些东西总是断开的：

- 想法和现实是断开的
- 蓝图和执行是断开的
- 对话和项目是断开的
- 一次经验和下一次复用是断开的

Ferry 把这些断裂统一理解为一种熵增问题。
它的目标就是：

- 对抗断裂
- 对抗熵增
- 对抗遗忘

也就是把高熵的混乱、未知和情绪，逐步转成低熵的有序、已知和产出。

## 对谁有用

Ferry 更适合这些人：

- 正在探索人机协作方法论的人
- 想把 AI 从“会聊天”推进到“会长期协作”的人
- 做编程、写作、法律、研究、个人成长等复杂系统工作的人
- 想把项目从一次性对话，推进到可继承、可进化结构的人

如果你关心的是：

- 人类主权和 AI 辅助如何分工
- 蓝图如何塑造现实
- 一套协作系统如何记录阻滞、复盘并进化

那 Ferry 会更适合你。

## 怎么开始

### 步骤 1：注入内核

将 [2、FerrySpec.world.md](./FerrySpec指导文档/2%E3%80%81FerrySpec.world.md) 的内容复制到 AI 助手的 System Prompt 或 Rules 中。

### 步骤 2：初始化世界

在项目根目录创建以下目录：

```text
Project_Root/
├── .ferry/          # [内核] FerrySpec.world.md 及扩展技能
├── blueprint/       # [意识] 蓝图、大纲、法律依据、理论支撑
├── reality/         # [物质] 实际产出、草稿、代码、证据
└── archive/         # [历史] 归档任务与进化日志
```

> 目录映射：编程语境中 `spec/` ↔ `blueprint/`，`src/` ↔ `reality/`

### 步骤 3：启动对话

| 状态 | 指令 |
| --- | --- |
| 一片空白 | `建立蓝图` |
| 正在攻坚 | `推进现实` |
| 陷入困境 | `辩证分析` |

## 使用效果

如果这套协议跑起来，它希望把协作推进到这种状态：

- 蓝图不再只停在脑子里，而会进入结构化目录
- 现实产出不再只是一时结果，而能被持续映射和修正
- 每一次阻滞都不只是卡住，而会进入日志并反哺下一次协作
- 人仍然保有价值裁决权，AI 则主要负责降熵与执行

它不是一个一次性 prompt，
而是一种让人机协作逐步稳定下来的工作方式。

## 核心结构

### 三位一体系统

| 组件 | 角色 | 职责 |
| --- | --- | --- |
| **Skill** | 大脑 / 基因 | 定规矩（宪法、能力、工具库） |
| **Flow** | 手脚 / 神经 | 干活（调用 Skill，执行操作，过闸门） |
| **Log** | 记忆 / 进化 | 长记性（记录阻滞，更新 Skill） |

> 一句话：Flow 在 Skill 的指导下撞墙，Log 把墙拆了，再更新 Skill 让下一次更顺畅。

### 五层逻辑

| 公理 | 内容 |
| --- | --- |
| **人类主权** | AI 负责逻辑推演、信息整理、方案生成等二次重复工作；Human 负责价值裁决、审美判断、最终拍板 |
| **熵减命令** | 系统的唯一目的是将高熵（混乱/未知）转化为低熵（有序/已知） |
| **双层世界观** | 蓝图层 = 应然；现实层 = 实然；偏差驱动行动 |
| **量变质变** | 系统在记录与积累后，分析问题归属，针对性修复潜在问题 |
| **做减法** | 如无必要，勿增实体；最简路径即最优路径 |

## 运行机制

### 状态调度（P0-P4）

| 阶段 | 状态特征 | 核心任务 |
| --- | --- | --- |
| **P0 混沌期** | 蓝图缺失 | 降熵：确立核心目标，建立蓝图 |
| **P1 规划期** | 蓝图初备 | 立法：拆解任务，制定验收标准 |
| **P2 执行期** | 任务活跃 | 造物：依据蓝图执行产出，物理落盘 |
| **P3 修正期** | 出现阻滞 | 辩证：分析矛盾，修正现实或蓝图 |
| **P4 进化期** | 目标达成 | 扬弃：归档复盘，提炼新技能 |

### 物理闸门

| 级别 | 类型 | 处理方式 |
| --- | --- | --- |
| 🟢 Low | 可逆操作 | 直接执行 |
| 🟡 Medium | 逻辑变更 | 一句话确认 |
| 🔴 High | 不可逆 / 高成本 | A/B 方案 + 风险评估 + 确认 |

### 进化逻辑

任务结束时，通过辩证法三段论写入 `archive/evolution.log`：

1. **肯定**：实现了什么物质成果？
2. **否定**：遇到了什么非预期阻滞？
3. **否定之否定**：沉淀了什么新规则或新技能？

## AI 局限性与责任边界

> **AI 无法承担后果。**
>
> 它能推演真假、罗列选项，但做错事不会疼，做对事无奖赏。最终按钮必须人类按。

| 局限类型 | 说明 | 影响 |
| --- | --- | --- |
| **认知局限** | 无直接经验，只能读文本；信息滞后，无法验证现实；会编造（幻觉） | 可能基于错误信息行动 |
| **责任局限** | 无法承担行动后果；无身体，无法物理试错；每次对话“失忆” | 不可逆操作必须人类决策 |

### 工程实践（三层分离）

| 层级 | AI 角色 | 操作类型 | 示例 |
| --- | --- | --- | --- |
| **AI 直接执行** | 执行者 | 可撤回 | 写代码、改代码、本地测试 |
| **AI 辅助** | 辅助者 | 需人类决策 | 提供选项化问题，人类选择 |
| **必须人类确认** | 等待者 | 不可逆 | 账号授权、部署、删数据、对外发布 |

一句话：

**AI 给建议，人类拍板担责。**
